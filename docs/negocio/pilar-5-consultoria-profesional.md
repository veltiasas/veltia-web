## 1. SERVICIOS

### 1.1 Relevamientos y Diagnósticos

#### 1.1.1 IA Discovery - Oportunidades de IA en el Negocio

**Descripción:** Workshop de 2-4 semanas para identificar casos de uso de IA aplicables al negocio del cliente.

**Proceso:**
1. **Kick-off (1 día):** Presentación de metodología, identificación de stakeholders
2. **Entrevistas (1-2 semanas):** Reuniones con áreas clave (operaciones, ventas, IT, RRHH)
3. **Análisis (1 semana):** Identificación de oportunidades, priorización por impacto/viabilidad
4. **Presentación (medio día):** Entrega de hallazgos y roadmap

**Entregables:**
- Informe ejecutivo (20-30 páginas)
- Lista de 5-10 casos de uso identificados
- Priorización por impacto (ROI estimado)
- Matriz de viabilidad técnica
- Roadmap de implementación (fases de 3-6 meses)
- Business case de cada caso de uso (top 3)

**Casos de uso típicos identificados:**
- Chatbot de atención al cliente
- Automatización de procesos repetitivos (RPA + IA)
- Análisis predictivo de ventas
- Clasificación automática de documentos
- Asistente interno para empleados

**ICP:** Empresas que saben que "necesitan IA" pero no saben por dónde empezar (50-500 empleados)

**Pricing:** USD 1.500 - 4.000

**Conversión típica:** 65% contratan implementación de al menos 1 caso de uso (Pilar 2)

**Líder:** Martín (facilitación) + Dino (viabilidad técnica)

**Ciclo de venta:** 7-15 días

#### 1.1.3 Diagnóstico de Transformación Digital

**Descripción:** Assessment de madurez digital de la organización con roadmap de transformación.

**Proceso:**
1. **Assessment (1 semana):** Evaluación de procesos, sistemas, cultura
2. **Benchmarking (1 semana):** Comparación con industria
3. **Roadmap (1 semana):** Plan de transformación priorizado
4. **Presentación (medio día):** Workshop con directivos

**Entregables:**
- Índice de madurez digital (score 1-100)
- Gap analysis (dónde estamos vs dónde deberíamos estar)
- Roadmap de transformación (3 años)
- Quick wins (0-6 meses)
- Business case de inversión

**Áreas evaluadas:**

| Dimensión | Evaluación |
|-----------|------------|
| **Procesos** | ¿Cuánto está digitalizado? |
| **Sistemas** | ¿Qué tan modernos son? |
| **Datos** | ¿Calidad, accesibilidad, analytics? |
| **Cultura** | ¿Resistencia al cambio? |
| **Skills** | ¿Capacidades del equipo? |

**ICP:** Pymes tradicionales (20-200 empleados) con procesos manuales

**Pricing:** USD 800 - 2.000

**Conversión típica:** 50% contratan al menos 1 servicio (Pilar 1 o 3)

**Líder:** Martín

**Ciclo de venta:** 7 días

### 1.2 Documentación Técnica

#### 1.2.1 Documentación de Código Legacy

**Descripción:** Documentación de sistemas sin documentación o con desarrolladores que ya no están.

**Proceso:**
1. **Análisis de código:** Reverse engineering del código fuente
2. **Mapeo de arquitectura:** Diagramas de componentes, flujos, base de datos
3. **Documentación funcional:** Qué hace cada módulo, endpoints, integraciones
4. **Runbooks:** Cómo ejecutar, deployar, troubleshoot

**Entregables:**
- Diagrama de arquitectura (C4 model)
- Documentación de endpoints/APIs (OpenAPI/Swagger)
- ERD de base de datos
- Runbooks de operación
- Guía de desarrollo (setup, deploy)

**ICP:** Empresas con sistema crítico sin documentación (rescate de apps abandonadas)

**Pricing:** USD 3.000 - 15.000 (según tamaño del sistema)

**Timeline:** 2-6 semanas

**Conversión típica:** 70% contratan mantenimiento o modernización (Pilar 3)

**Líder:** Dino (análisis) + documentador técnico (freelance)

**Ciclo de venta:** 7-15 días

---

#### 1.2.2 Documentación de Procesos de Negocio

**Descripción:** Mapeo y documentación de procesos críticos de la empresa (no IT, sino negocio).

**Proceso:**
1. **Relevamiento (1-2 semanas):** Entrevistas con responsables de proceso
2. **Modelado (1 semana):** Diagramas BPMN
3. **Documentación (1 semana):** Descripción detallada de cada actividad
4. **Validación (1 semana):** Revisión con stakeholders

**Entregables:**
- Diagramas de proceso (BPMN)
- Descripción de actividades
- Matriz RACI (responsables)
- Identificación de cuellos de botella
- Recomendaciones de mejora

**Casos de uso:**
- Empresa certificando ISO 9001 (requiere procesos documentados)
- Empresa con alta rotación (conocimiento se pierde)
- Empresa preparándose para crecer (estandarización)

**ICP:** Pymes en crecimiento que necesitan estandarizar

**Pricing:** USD 2.000 - 8.000 (según cantidad de procesos)

**Conversión típica:** 50% contratan automatización (Pilar 1 o 2)

**Líder:** Martín (análisis) + analista de procesos (freelance)

**Ciclo de venta:** 15-30 días

### 1.3 Desarrollos a Medida

#### 1.3.1 MVP de Software a Medida

**Descripción:** Desarrollo rápido de MVP para validar idea de producto/servicio.

**Metodología:**
- **Design Sprint (1 semana):** Definición de features core
- **Desarrollo (4-8 semanas):** Implementación de MVP
- **Testing (1 semana):** Validación con usuarios
- **Deploy (1 semana):** Lanzamiento en producción

**Stack:**
- Backend: FastAPI (Python) / NestJS (TypeScript)
- Frontend: React + Next.js
- Base de datos: PostgreSQL + Supabase
- Deployment: Vercel / AWS

**ICP:** Startups, emprendedores, empresas validando nuevas líneas de negocio

**Pricing:** USD 8.000 - 30.000 (según complejidad)

**Timeline:** 6-10 semanas

**Conversión típica:** 80% contratan mantenimiento/evolución continua

**Líder:** Dino (desarrollo) + Martín (PM)

**Ciclo de venta:** 15-30 días

#### 1.3.2 Integraciones a Medida

**Descripción:** Desarrollo de integraciones entre sistemas del cliente.

**Casos de uso:**
- Integrar CRM con ERP
- Integrar plataforma eCommerce con sistema de inventario
- Integrar sistema de RRHH con plataforma de firma digital
- Sincronizar datos entre sistemas legacy y modernos

**Proceso:**
1. **Análisis (1 semana):** Mapeo de sistemas, APIs, datos
2. **Diseño (1 semana):** Arquitectura de integración
3. **Desarrollo (2-6 semanas):** Implementación
4. **Testing (1 semana):** Validación de datos
5. **Deploy (1 semana):** Producción + monitoreo

**Tecnologías:**
- APIs REST, GraphQL, SOAP
- ETL: Apache Airflow, custom Python scripts
- Middleware: RabbitMQ, Kafka
- iPaaS: Zapier, Make (si aplica)

**ICP:** Empresas con sistemas desconectados (datos duplicados, procesos manuales)

**Pricing:** USD 3.000 - 20.000 (según complejidad)

**Timeline:** 4-10 semanas

**Líder:** Dino (desarrollo) + Ale (integración infra)

**Ciclo de venta:** 15-45 días

## 2. ESTRATEGIA DE FUNNEL

### Modelo de Conversión

```
┌──────────────────────┐
│   CONSULTORÍA        │ ← Entrada (bajo ticket, bajo riesgo)
│ (USD 1K-5K, 1-4 sem) │
└──────────┬───────────┘
           │ 60-70% convierten
┌──────────▼───────────┐
│   PROYECTO           │ ← Implementación (ticket medio)
│ (USD 10K-50K, 2-6m)  │
└──────────┬───────────┘
           │ 40-50% convierten
┌──────────▼───────────┐
│   RECURRENTE         │ ← Relación long-term (MRR)
│ (USD 1K-5K/mes)      │
└──────────────────────┘
```

### Tácticas de Conversión

| Etapa | Táctica |
|-------|---------|
| **Consultoría → Proyecto** | Incluir "fase 1 de implementación" en propuesta de consultoría |
| **Proyecto → Recurrente** | Ofrecer mantenimiento/soporte desde kick-off |
