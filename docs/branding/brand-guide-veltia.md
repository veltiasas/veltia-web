## 1. NOMBRE

### Uso Correcto

- El nombre **VELTIA** se escribe siempre en **mayusculas**
- No usar: Veltia, veltia, VeLTiA, ni ninguna otra variante
- En contextos legales: VELTIA SAS

### Tagline

**"Tecnologia que transforma"**

- Siempre en minusculas (salvo inicio de oracion)
- Separado del nombre con un guion largo o en linea aparte
- No modificar ni parafrasear

### Ejemplos de Uso

| Contexto | Formato |
|----------|---------|
| Header web | VELTIA |
| Subtitulo | VELTIA -- Tecnologia que transforma |
| Email signature | VELTIA \| Tecnologia que transforma |
| Documento formal | VELTIA SAS |
| Redes sociales | @veltia.tech (sugerido) |

| Rol | Color | Hex | RGB | Uso |
|-----|-------|-----|-----|-----|
| **Primario** | Azul oscuro | `#003366` | 0, 51, 102 | Headers, fondos principales, texto destacado |
| **Secundario** | Gris plata | `#C0C0C0` | 192, 192, 192 | Bordes, separadores, texto secundario |

### Variantes

| Base | Light | Dark | Uso |
|------|-------|------|-----|
| `#003366` | `#004C99` | `#00264D` | Hover states, gradientes |
| `#C0C0C0` | `#E0E0E0` | `#808080` | Bordes sutiles, disabled |

### Reglas de Combinacion

- **Fondo oscuro** (`#003366`): texto blanco (`#FFFFFF`) o amarillo (`#FFD700`)
- **Fondo claro** (`#FAFAFA`): texto oscuro (`#1A1A2E`)
- **CTAs**: fondo amarillo (`#FFD700`) con texto oscuro (`#1A1A2E`)
- **Links**: azul primario (`#003366`), hover azul claro (`#004C99`)
- Ratio de contraste minimo: 4.5:1 (WCAG AA)

## 4. TONO DE VOZ

### Principios

| Principio | Descripcion |
|-----------|-------------|
| **Directo** | Ir al grano. Sin rodeos ni corporate speak |
| **Moderno** | Lenguaje de startup. Fresco pero profesional |
| **Tecnico accesible** | Hablar de tecnologia sin jerga innecesaria |
| **Confiable** | Datos concretos, no promesas vacias |
| **Voseo** | Usar "vos" en lugar de "tu" (mercado argentino) |

### Do's

- Usar voseo: "Reducis costos", "Transformas tu negocio"
- Ser especifico: "Reducimos 30-50% tu factura cloud" (no "ahorra mucho")
- Usar numeros: "En 4 semanas", "ROI de 6 meses"
- Hablar de resultados, no de features
- Usar verbos de accion: "Automatiza", "Migra", "Escala"

### Don'ts

- No usar "estimado cliente" ni lenguaje corporativo formal
- No usar emojis en documentos formales
- No prometer sin datos: evitar "el mejor", "lider del mercado"
- No usar anglicismos innecesarios (excepto terminos tecnicos estandar)
- No exagerar: si el ahorro es 30%, no decir "hasta 50%"
- No tutear: siempre voseo

### Ejemplos

| Mal | Bien |
|-----|------|
| "Estimado, le informamos que..." | "Te contamos como funciona" |
| "Somos lideres en soluciones digitales" | "Ayudamos a 15+ empresas a reducir costos cloud 35%" |
| "Nuestra plataforma leveragea AI para..." | "Usamos IA para automatizar tus procesos" |
| "No dudes en contactarnos" | "Escribinos y arrancamos" |

## 5. ICONOGRAFIA Y ESTILO VISUAL

### Simbolos Permitidos

| Simbolo | Uso |
|---------|-----|
| `->` | Flujos, navegacion, CTAs |
| `*` | Bullets, listas |
| `(check)` | Confirmaciones, features incluidas |
| `--` | Separadores, rangos |
| `|` | Separadores en linea |

### Estilo de Imagenes

- **Ilustraciones:** Estilo flat, geometrico, colores de la paleta
- **Fotos:** Minimas. Si se usan: ambientes de trabajo tech, equipos diversos
- **Graficos:** Limpios, colores de la paleta, sin decoracion excesiva
- **Iconos:** Linea fina (stroke), estilo Lucide o Phosphor Icons

### Espaciado

- Padding seccion: 64px (4rem) vertical
- Gap entre elementos: 24px (1.5rem)
- Margen de contenido: max-width 1200px, centrado
- Border radius: 8px (botones), 12px (cards), 16px (secciones)
